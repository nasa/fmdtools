Examples

This repository provides several resources to get familiar with fmdtools.

To get started with fmdtools, it is reccomended to start with the Intro to fmdtools workshop, which provides a basic high-level overview of modelling and simulation in fmdtools, with a corresponding model and example notebook to fill out. To start the workshop, first download the workshop slides (Intro to fmdtools.pptx), tutorial model (ex_pump.py), and Unfilled Notebook ex_pump.py. If you cloned fmdtools, you can just navigate to these files in the repository–they are in a directory called example_pump. Then, follow along with the slides and the (see: filled-in notebook).

After completing the workshop, it can be helpful to run through the following notebooks to better understand fmdtools modelling, simulation, and analysis basics as well as more advanced features and use-cases:

  • Defining and Visualizing fmdtools Model Structures is helpful for understanding how a given model simulates over time. It covers the class fmdtools.analyze.graph.Graph and containing methods.

  • fmdtools Paper Demonstration is helpful for understanding network/static/dynamic/hierarchical model types, covering:

    • The fmdtools.sim.networks module.

    • Basic simulation of dynamic and static models using methods in fmdtools.sim.propagate and usage of class fmdtools.define.SampleApproach for fault sampling

    • Analysis using Basic analysis/results processing capabilities

  • Pump Example Notebook is helpful for understanding the breadth of fmdtools plotting, tabulation, and visualization capabilities. It covers:

    • A variety of graphing usecases in fmdtools.analyze.graph functions which enable viewing different graph types, simulation results at individual times, and overall model statistics/results.

    • fmdtools.analyze.tabulate functions for viewing simulation results over time and summarizing run information

    • Save/load using the dill package

  • Defining Fault Sampling Approaches in fmdtools covers how to set up a fault sampling approach and use it to simulate a large number of hazardous scenarios in a model. This includes:

    • Adding fault and operational modes to Model functions using the method fmdtools.define.Block.assoc_modes() and explanation of the key_phases_by and exclusive options.

    • Using fmdtools.analyze.result.History.get_modephases() to setting a fmdtools.define.SampleApproach up which samples individual faults based on the phases of the model and/or function defined by their operational modes.

    • Using fmdtools.analyze.plot.phases() to visualize the phases and modes of a model over time and fmdtools.analyze.plot.samplecosts() to visualize the consequences of each fault scenario in the approach within each phase.

    • Using the defaultsamp option and fmdtools.define.SampleApproach.prune_scenarios() in fmdtools.define.SampleApproach to control how many time-steps in each phase are sampled (and when).

  • Using Parallel Computing in fmdtools covers how to reduce computational costs for computationally-expensive simulation use-cases (e.g., sampling large numbers of fault scenarios or running complex models with large numbers of timesteps). It covers:

    • Using process pools as arguments to fmdtools.sim.propagate methods to speed up simulation. A comparison of process pools from different external python packages are provided.

    • Different options for history tracking and staged execution which can reduce computational costs when desired.

    • Profiling models with cProfile and pycallgraph2 to discover what parts are most computationally-expensive.

  • Defining Nominal Approaches in fmdtools , which covers simulating the model at different parameters in nominal/faulty scenarios. This includes:

    • Setting up a nominal parameter sampling approach using fmdtools.define.NominalApproach and simulating it with fmdtools.sim.propagate.nominal_approach() and fmdtools.sim.propagate.nested_approach() methods for nominal and faulty simulations.

    • Using analysis functions like fmdtools.analyze.tabulate.nominal_vals_1d() and fmdtools.analyze.plot.nominal_factor_comparison() to visualize quantities of interest for the simulation over a range of nominal parameters.

    • Using analysis functions like fmdtools.analyze.tabulate.resilience_factor_comparison() and fmdtools.analyze.plot.resilience_factor_comparison() to visualize resilience metrics of the model to a set of fault modes over a range of nominal parameters.

  • Stochastic Modelling in fmdtools , which covers defining and simulating stochastic models–models with random internal behaviors. This includees:

    • Setting up random states in functions using fmdtools.define.Block.assoc_rand_state(), fmdtools.define.Block.set_rand(), and fmdtools.define.Block.to_default().

    • Simulating stochastic models using the run_stochastic parameter in fmdtools.sim.propagate functions, as well as setting up a fmdtools.define.NominalApproach with multiple seeds to run a set of random simulations.

    • Using fmdtools.analyze.plot.mdlhists() to visualize the results of multiple stochastic simulations over time, and analyze quantities of interest using fmdtools.analyze.tabulate.nested_stats(), fmdtools.analyze.tabulate.resilience_factor_comparison()

A number of more advanced techniques have been developed using the rover model in example_rover. Rover Setup Notebook , introduces this model, which is then used to:

  • Showcase fmdtools human factors modelling capabilities (action sequence graphs, performance shaping factors, etc.) in HFAC Analyses and IDETC_Human_Paper_Analysis

  • Show how degradation models can be used to sample resilience model parameters in Degradation Modelling Notebook

  • Show how fault scenarios can be exhaustively generated to augment hazard identification in Rover Mode Notebook

  • Show a method for searching the faulty state-space to find a unique set of hazardous scenarios in Search Comparison

  • Demonstrate the optimization of parameters over a set of fault scenarios using ProblemInterface in Rover Response Optimization

The following notebooks showcase the optimization interfaces in fmdtools.sim.search:

  • Optimization shows some of the basics of working with the fmdtools.sim.search.ProblemInterface class for optimization.

  • AST Sampling shows how fmdtools models called from the AdaSress Julia package to leverage the adaptive stress testing methodology using the fmdtools.sim.search.DynamicInterface class.

  • Multirotor Optimization shows how the design, operations, and contingency management of a system can be co-optimized with the fmdtools.sim.search.ProblemInterface class.

  • Tank Optimization shows how design and contingency management of a system can be co-optimized with the fmdtools.sim.search.ProblemInterface class, as well as external solvers.

There are also two other example models which demonstrate specialized use-cases:

  • Hold-up Tank Model uses the fmdtools.define.SampleApproach class to model human interactions with the modelled system.

  • EPS Example Notebook shows a simple static modelling use-case.
