Glossary

Function

A piece of functionality in a system which has its own defined behavior, modes, and flow connections, and may be further instantiated by a component architecture or action sequence graph. In general, functions are the main major building block of a model defining how the different pieces of the system behave. Functions in fmdtools are specified by extending the fmdtools.define.FxnBlock class.

Flow

A data structure which connects functions–traditionally energy, material, or signal. Instantiated using the fmdtools.define.Model.add_flow() method (for easily-specified flows) or fmdtools.define.Flow (for custom flows).

Internal Flow

A flow object that is internal to a fmdtools.define.FxnBlock which is used to connect fmdtools.define.Action s and fmdtools.define.Component s

Model

A simulation that defines system behavior. Models contain functions and flows, their graph connections, parameters related to the simulation configuration, as well as methods for classifying simulations. Models are specified using the fmdtools.define.Model: class.

Behavior

How the states of a system unfold over time, including in the various mode s it may encounter. Defined in Function s, Component s, and Action s using fmdtools.define.Block.behavior(), fmdtools.define.Block.static_behavior(), fmdtools.define.Block.dynamic_behavior(), and fmdtools.define.Block.condfaults()

Graph

A view of simulation construct connections and/or relationships embodied by the fmdtools.analyze.graph.Graph class (which uses networkx to represent the structure itself).

Component

A physical that embodies specific behavior for a function. May have mode s and behavior s of its own. Specified by extending the fmdtools.define.Component class.

Component Architecture

The physical embodiment of a function that encompasses multiple Component s.

Mode

Discrete modifications of a behavior specified as strings in fmdtools.define.Block.assoc_modes(). Often used to control if/else statements in a fmdtools.define.Block.behavior() method.

Fault Mode

Undesired mode, which leads to hazardous behavior. For example, a lamp may have “burn-out” and “flicker” modes

Operational Mode

Defined mode that the system progresses through as a part of its desired functioning. For example, a light switch may be in “on” and “off” modes.

Action Sequence Graph

An instance of the define.block.ASG which embodies a (human or autonomous) agent ‘s sequence of tasks which it performs to accomplish a certain function.

Agent

An actor which controls behaviors in a system. May be modelled as a function.

Environment

The uncontrolled aspect of a system which may effect system inputs and behaviors. May be modelled as a function.

Action

A specific task to be performed by an agent used to represent human/autonomous operations. May be specified by extending the fmdtools.define.Action class and added to a fmdtools.define.FxnBlock using fmdtools.define.FxnBlock.add_act().

Rate

The expected occurence (frequency) of a given mode, which may be specified in a number of ways using fmdtools.define.FxnBlock.assoc_modes().

Cost

A metric used to define severity of a scenario. While cost is defined in a monetary sense, it should often be defined holistically to account for indirect costs and externalities (e.g., safety, disruption, etc). One of the default outputs from fmdtools.define.Model.find_classification().

Expected Cost

A metric used to define risk of a scenario, calculated my multiplying the rate and cost.

Endclass

The end-state classification given from fmdtools.define.Model.find_classification().

Scenario

A specific set of inputs to a simulation, including fmdtools.define.Model parameters, Fault Mode s, and Disturbances.

Disturbances

A specific sequence of variable values over time which may modify system behavior.

Approach

A set of :term:` scenario` s to simulate a model over to represent certain hazards or parameters of interest. May be generated using fmdtools.define.SampleApproach for fault modes or :class:’NominalApproach’ for nominal parameters.

Nested Approach

The result of simulating a fault sampling Approach (fmdtools.define.SampleApproach) within a nominal Approach (fmdtools.define.NominalApproach). Created in fmdtools.sim.propagate.nested_approach()

Static Propagation

The undirected propagation of model behaviors within a timestep. Defined for each function using fmdtools.define.FxnBlock.static_behavior(), which may run multiple times in a timestep until behavior has converged.

Dynamic Propagation

The progression of model states over time. Defined for each function using fmdtools.define.FxnBlock.dynamic_behavior(), which runs once per timestep.

Propagation

The simulation of fmdtools.define.Model behavior s, including the passing of flow s between function s and the progression of model states over time.

Resilience

The expectation of defined performance metrics over time over a set of hazardous :term:` scenario` s, often defined in terms of the deviation from their nominal values.

End-state

The state of a fmdtools.define.Model at the final time-step of a simulation.

FMEA

A table outlining the risks of hazardous :term:` scenario` s in terms of their rate, severity, and expected risk. By default, the fmdtools.analyze.tabulate module produces cost-based FMEAS, with the metrics of interest being rate, cost, and expected cost, however these functions can be tailored to the metrics of interest.

Behavior Over Time

How a the states of a system unfold over time. Defined using behavior.

Model History

A history of model states over a set of timesteps. Returned in fmdtools as a nested dictionary from methods in fmdtools.sim.propagate.
