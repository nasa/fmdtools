Examples

This repository provides several resources to get familiar with fmdtools.

To get started with fmdtools, it is reccomended to start with the Intro to fmdtools workshop, which provides a basic high-level overview of modelling and simulation in fmdtools, with a corresponding model and example notebook to fill out. To start the workshop, first download the workshop slides (Intro to fmdtools.pptx), tutorial model (ex_pump.py), and Unfilled Notebook ex_pump.py. If you cloned fmdtools, you can just navigate to these files in the repository–they are in a directory called example_pump. Then, follow along with the slides and the (see: filled-in notebook).

After completing the workshop, it can be helpful to run through the following notebooks to better understand fmdtools modelling, simulation, and analysis basics as well as more advanced features and use-cases:

  • Defining and Visualizing fmdtools Model Structures is helpful for understanding how a given model simulates over time. It covers the methods:

    • fmdtools.resultdisp.graph.show()

    • fmdtools.resultdisp.graph.exec_order()

    • fmdtools.resultdisp.plot.graph_order()

  • fmdtools Paper Demonstration is helpful for understanding network/static/dynamic/hierarchical model types, covering:

    • The fmdtools.faultsim.networks module.

    • Basic simulation of dynamic and static models using methods in fmdtools.faultsim.propagate and usage of class fmdtools.modeldef.SampleApproach for fault sampling

    • Analysis using fmdtools.resultdisp.process.hists(), fmdtools.resultdisp.plot.mdlhistvals() fmdtools.resultdisp.tabulate.fullfmea(), and fmdtools.resultdisp.tabulate.simplefmea()

  • Pump Example Notebook is helpful for understanding the breadth of fmdtools plotting, tabulation, and visualization capabilities. It covers:

    • A variety of graphing usecases in fmdtools.resultdisp.graph functions which enable viewing different graph types, simulation results at individual times, and overall model statistics/results.

    • fmdtools.resultdisp.tabulate functions for viewing simulation results over time and summarizing run information

    • Save/load using the dill package

  • Defining Fault Sampling Approaches in fmdtools covers how to set up a fault sampling approach and use it to simulate a large number of hazardous scenarios in a model. This includes:

    • Adding fault and operational modes to Model functions using the method fmdtools.modeldef.Block.assoc_modes() and explanation of the key_phases_by and exclusive options.

    • Using fmdtools.resultdisp.process.modephases() to setting a fmdtools.modeldef.SampleApproach up which samples individual faults based on the phases of the model and/or function defined by their operational modes.

    • Using fmdtools.resultdisp.plot.phases() to visualize the phases and modes of a model over time and fmdtools.resultdisp.plot.samplecosts() to visualize the consequences of each fault scenario in the approach within each phase.

    • Using the defaultsamp option and fmdtools.modeldef.SampleApproach.prune_scenarios() in fmdtools.modeldef.SampleApproach to control how many time-steps in each phase are sampled (and when).

  • Using Parallel Computing in fmdtools covers how to reduce computational costs for computationally-expensive simulation use-cases (e.g., sampling large numbers of fault scenarios or running complex models with large numbers of timesteps). It covers:

    • Using process pools as arguments to :module:`fmdtools.faultsim.propagate` methods to speed up simulation. A comparison of process pools from different external python packages are provided.

    • Different options for history tracking and staged execution which can reduce computational costs when desired.

    • Profiling models with cProfile and pycallgraph2 to discover what parts are most computationally-expensive.

  • Defining Nominal Approaches in fmdtools , which covers simulating the model at different parameters in nominal/faulty scenarios. This includes:

    • Setting up a nominal parameter sampling approach using fmdtools.modeldef.NominalApproach and simulating it with fmdtools.faultsim.propagate.nominal_approach() and fmdtools.faultsim.propagate.nested_approach() methods for nominal and faulty simulations.

    • Using analysis functions like fmdtools.resultdisp.tabulate.nominal_vals_1d() and fmdtools.resuldisp.plot.nominal_factor_comparison() to visualize quantities of interest for the simulation over a range of nominal parameters.

    • Using analysis functions like fmdtools.resuldisp.tabulate.resilience_factor_comparison() and fmdtools.resuldisp.plot.resilience_factor_comparison() to visualize resilience metrics of the model to a set of fault modes over a range of nominal parameters.

  • Stochastic Modelling in fmdtools , which covers defining and simulating stochastic models–models with random internal behaviors. This includees:

    • Setting up random states in functions using fmdtools.modeldef.Block.assoc_rand_state(), fmdtools.modeldef.Block.set_rand(), and fmdtools.modeldef.Block.to_default().

    • Simulating stochastic models using the run_stochastic parameter in fmdtools.faultsim.propagate functions, as well as setting up a fmdtools.modeldef.NominalApproach with multiple seeds to run a set of random simulations.

    • Using fmdtools.resultdisp.plot.mdlhists() to visualize the results of multiple stochastic simulations over time, and analyze quantities of interest using fmdtools.resultdisp.tabulate.nested_stats(), fmdtools.resultdisp.tabulate.resilience_factor_comparison()

There are also two other example models which demonstrate specialized use-cases:

  • Hold-up Tank Model uses the fmdtools.modeldef.SampleApproach class to model human interactions with the modelled system.

  • EPS Example Notebook shows a simple static modelling use-case.
